Double Diamond Model (Discover → Define → Develop → Deliver)

Discover: Looked at how users interact with the Google homepage and identified needs like simplicity and quick access.

Define: Focused on clarity, usability, and accessibility.

Develop: Created a cleaner layout with structured navigation, tooltips, and search options.

Deliver: Produced the redesigned homepage in GooglePage.html.



Conceptual Models

Metaphors: Used the search symbols such as magnifying glass, microphone, and camera.

Assumptions: Users want an easy page that they can navigate.

User Mental Models: It looks just like the original Google Page, but it is easier to navigate.



People-Centered Design Principles

Clarity and simplicity: Focus on the search box and no distractions.

Accessibility: Added labels and voice support.

Feedback: Voice output confirms interactions.

Consistency: Keeps the iconic Google layout and branding.



Interaction Types and Usability Criteria

Interaction Types: Typing into the search bar, clicking icons, using voice input.

Usability Criteria:

Effectiveness: Main task (search bar) is there.

Efficiency: Easy access to Gmail, Images, and settings.

Satisfaction: Clean and familiar design.

Learnability: Tools and simple structure make it easy to use.



Application in Redesign

Tooltips explain icons.

Added voice and image search.

Navigation is grouped in clear areas (header and footer).

Voice feedback provides accessibility support.

Layout works across devices.



Why These Principles Improve UX

Following user mental models reduces confusion.

People-centered design increases usability for more users.

The Double Diamond process ensures changes are guided by user needs.

Consistency maintains trust and ease of use.



Assumptions and Claims

Users are familiar with the Google homepage.

Users benefit from more than one way to search.

Tooltips and feedback improve confidence and reduce mistakes.



Problems Being Solved

Icons are made clearer through tooltips.

Accessibility is supported through ARIA labels and voice feedback.

Users receive better guidance on how to interact with the page.



Evidence Supporting Design Choices

Google’s existing focus on minimalism and speed influenced the design.

Human-Computer Interaction principles support input for accessibility.
